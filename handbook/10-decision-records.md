# 决策记录与责任归属

## 页面定位

本页定义组织如何记录决定、区分建议与承诺，并明确负责人。它关注影响后续行动的决策如何被保存和复核。

## 内容梗概

本页应帮助团队避免“大家好像同意了”“会议里说过”“负责人不清楚”等问题。每个重要决定都应说明背景、选项、取舍、最终决定、影响范围、负责人和复核时间。

## 建议大纲

### 什么需要决策记录

- 会影响多个团队或长期执行方式。
- 会改变项目方向、范围、资源或里程碑。
- 有多个可行选项且存在明显取舍。
- 后续成员需要理解为什么这样做。

### 区分四类表达

- 讨论：正在探索，还没有结论。
- 建议：提出者认为可行，但尚未被采纳。
- 决策：负责人已经选择方案。
- 承诺：某人或团队明确接受行动责任。

### 决策记录结构

- 标题。
- 日期。
- 决策人或负责人。
- 背景。
- 目标。
- 备选方案。
- 取舍分析。
- 最终决定。
- 影响范围。
- 后续行动。
- 复核时间。

### 责任归属

- 每项长期机制应有 Owner。
- 跨团队事项应有最终负责推动的人。
- Owner 负责保持信息当前有效，不代表独自完成全部工作。

### 决策变更

- 改变决定时应说明新事实或约束。
- 不应悄悄覆盖旧决策而不保留变更原因。
- 旧决策应链接到新决策或归档说明。

### Agent 可检查性

- Agent 可以检查是否存在负责人、日期、选项、结论和行动项。
- Agent 可以提示讨论和决定混杂。
- Agent 不应代替负责人判断哪个方案正确。

## 后续需补充

- 决策记录的最小字段示例。
- 决策等级和复核周期。
- 决策变更示例。
- 不同平台中决策记录的存放位置建议。
