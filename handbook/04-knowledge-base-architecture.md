# 信息源与知识库架构

## 页面定位

本页说明组织级知识库应如何构建。它关注组织如何长期保存可复用知识，而不是某个项目文件夹或个人笔记如何摆放。

## 内容梗概

本页应定义单一事实源、页面归属、目录层级、知识生命周期和跨系统链接方式。读者应能据此判断一条信息应进入 Handbook、项目空间、任务系统、决策记录、平台落地说明还是研究材料。

## 建议大纲

### 组织级知识库的目标

- 让组织运行方式可被新人、跨团队成员和 Agent 理解。
- 让决策背景和执行结果可追踪。
- 降低对口头传承、聊天搜索和个人记忆的依赖。

### 推荐信息分层

- Handbook：组织协作方式和长期规范。
- Team Manual：团队职责、服务边界和工作节奏。
- Project Space：项目背景、目标、里程碑、状态和风险。
- Decision Records：长期影响后续行动的决定。
- Task System：行动项、负责人、状态和交付物。
- Knowledge Articles：问题导向的知识条目和案例。
- Research：调查材料和外部参考。

### 单一事实源

- 同一项规范只能有一个权威页面。
- 重复出现的内容应链接引用，而不是复制维护。
- 如果两个页面冲突，应以更接近规范来源的页面为准，并修正文档。

### 页面归属

- 每个长期页面应有 Owner。
- Owner 负责维护页面可用性，不等于独自执行页面中的所有工作。
- 无 Owner 页面应被标记、迁移或归档。

### 生命周期

- 创建：说明为何需要新页面。
- 维护：定期复核事实、链接和适用范围。
- 合并：重复内容应归并到权威页面。
- 废弃：旧页面应保留迁移路径。
- 归档：历史内容应明确不再作为当前事实。

### 跨系统链接

- 任务应链接到背景文档和交付物。
- 会议纪要应链接到决策记录和行动项。
- 项目空间应链接到里程碑、状态更新和风险记录。
- IM 中的重要结论应迁移到长期载体后再分享链接。
