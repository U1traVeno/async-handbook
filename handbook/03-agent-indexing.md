# Agent 可检索与自动化规范

## 页面定位

本页定义 Handbook 和组织知识库如何对 Agent 友好，以及 Agent 在自动检查、引用规范和提出建议时的边界。

## 内容梗概

本页应说明面向 Agent 的设计不是把文档变成配置文件，而是让人类可读文本拥有稳定标题、元数据、链接、术语和权威来源。Agent 可以辅助发现缺失和风险，但不能代替负责人作业务判断。

## 建议大纲

### Agent 友好的信息结构

- 标题稳定。
- 页面类型明确。
- Owner、状态、更新时间和复核时间可见。
- 相关页面显式链接。
- 术语与术语表一致。

### 推荐元数据

- `type`：页面类型。
- `owner`：维护负责人。
- `status`：页面状态。
- `updated_at`：最后更新时间。
- `review_at`：下次复核时间。
- `audience`：主要读者。
- `tags`：主题标签。
- `source_of_truth`：是否为权威来源。

### 可检查性分级

- 确定性检查：结构、字段、链接和状态是否存在。
- 启发式检查：上下文是否充分、标题是否清楚、结论是否可追踪。
- 人工判断：业务决策、优先级、风险接受度和组织取舍。

### Agent 引用规范

- 引用具体页面和标题。
- 说明观察到的证据。
- 区分确定问题和潜在风险。
- 提供可执行建议。
- 在上下文不足时说明不确定性。

### 自动化建议

- 过期页面提醒。
- 无 Owner 页面提醒。
- 无长期入口的 IM 群提醒。
- 里程碑复核提醒。
- 会议无纪要提醒。
- 决策记录缺少负责人或行动项提醒。

### Agent 不应做什么

- 不应创造 Handbook 中不存在的要求。
- 不应把启发式判断伪装成确定违规。
- 不应评价个人品格或文化分数。
- 不应因无法访问受限上下文就断言信息不存在。
