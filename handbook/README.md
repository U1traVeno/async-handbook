# Handbook

本目录是 Async Collaboration Handbook 的正文入口，供人类与 AI 共同阅读。

Handbook 的正文描述当前采用的协作做法，并解释为什么这样做、适用边界是什么，以及不同场景下应如何判断。页面围绕具体问题自然组织，不使用统一模板；平台落地说明不得新增规范正文中不存在的义务。

## 当前内容

- [异步协作介绍](00-about-async-work.md)：介绍异步协作制度的基本概念、优势，以及落实异步协作的简单建议。
- [异步协作原则](01-async-principles.md)：定义 Handbook 的上位协作原则，其他页面描述最终都应回到这些原则解释原因和边界。
- [Handbook 编写规范](02-handbook-writing-guide.md)：说明 Handbook 本身应如何编写、审阅和维护。
- [Agent 如何使用 Handbook](03-agent-indexing.md)：说明 Agent 如何查阅、引用和审阅规范，以及如何避免机械检查和越权判断。
- [信息源与知识库架构](04-knowledge-base-architecture.md)：说明如何根据信息用途、权限、生命周期和权威来源组织知识。
- [协作文档怎样帮助工作继续](05-documentation-standards.md)：说明普通文档如何按读者目标和风险提供适量上下文。
- [任务发布与进度追踪](06-task-management.md)：定义任务说明、状态更新、受阻处理和交付闭环。
- [如何选择沟通渠道](07-communication-channels.md)：说明如何根据信息用途选择和切换文档、任务、IM、私聊及同步沟通。
- [项目计划与里程碑](08-project-milestones.md)：定义里程碑如何服务风险管理和决策检查。
- [会议如何服务异步协作](09-meetings.md)：说明何时同步沟通值得投入，以及如何保护参与者的共同注意力。
- [决策记录与责任归属](10-decision-records.md)：定义决策、建议、承诺和负责人的记录方式。
- [组织运行节奏](11-operating-cadence.md)：说明如何根据变化速度、风险和受众建立反馈循环。
- [入职、交接与知识传承](12-onboarding-handoffs.md)：定义新成员获取上下文和角色变化时的知识迁移。
- [术语表](13-glossary.md)：维护本手册使用的关键术语。
- [研究材料](research/)：保存会长期影响本项目设计的调查材料；其中内容不是规范正文。

## 阅读顺序

首次参与本项目时，建议先阅读仓库根目录的 [README](https://github.com/U1traVeno/async-handbook/blob/main/README.md)，再阅读本页、[异步协作原则](01-async-principles.md)、[Handbook 编写规范](02-handbook-writing-guide.md)和[Agent 如何使用 Handbook](03-agent-indexing.md)。

## 如何提出修改

有具体文本可供审阅时，应直接提出修改；问题尚不清楚、影响跨越多个章节或存在明显迁移风险时，可以先展开讨论。所有已经形成的改变都应回到 Handbook 正文，而不是停留在聊天或会议中。
