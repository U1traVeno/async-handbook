# Handbook

本目录是 Async Collaboration Handbook 的正文入口，供人类与 AI 共同阅读。

Handbook 的正文应描述当前采用的协作做法，并解释为什么这样做、适用边界是什么、有哪些示例，以及 Agent 可以如何审阅。平台落地说明也可以放在 Handbook 中，但必须和平台无关的规范正文隔离，不能新增正文里没有的义务。

当前编号页面先用于确定各章节的收录边界和展开顺序。后续将单页展开为正式正文时，应按[Handbook 编写规范](02-handbook-writing-guide.md)围绕具体问题自然组织内容，不把当前草案结构当作固定格式。

## 阅读顺序

首次参与本项目时，建议先阅读仓库根目录的 [README](../README.md)，再阅读本页、[异步协作原则](01-async-principles.md)、[Handbook 编写规范](02-handbook-writing-guide.md)和[Agent 可检索与自动化规范](03-agent-indexing.md)。

准备进入具体协作场景时，再从[信息源与知识库架构](04-knowledge-base-architecture.md)继续阅读。需要理解 GitLab Handbook 对本项目的启发时，再阅读[研究材料](research/)。

## 如何提出修改

小改动应直接修改文本。大改动应先说明问题、影响范围、迁移方式和未决问题。所有改变都应回到 Handbook 正文，而不是停留在聊天或会议中。
