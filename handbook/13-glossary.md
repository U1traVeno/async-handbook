# 术语表

本页解释 Async Handbook 中反复出现且容易产生不同理解的术语。定义帮助读者找到共同语言，不替代相关规范正文。

## Agent

能够读取上下文并协助分析、生成内容或执行操作的 AI 系统。Agent 的能力取决于获得的上下文、工具和权限，不能被视为天然了解组织背景。参见[Agent 如何使用 Handbook](03-agent-indexing.md)。

## 异步协作

参与者不需要同时在线，也能依靠共享上下文独立推进并交接工作的协作方式。异步协作强调工作连续性，不等同于只使用文字或取消会议。参见[异步协作原则](01-async-principles.md)。

## 同步沟通

参与者在同一时间窗口内直接交流，例如当面讨论、电话、视频会议和实时结对。同步沟通适合需要快速来回、共同观察或情绪感知的场景。参见[会议如何服务异步协作](09-meetings.md)。

## Handbook

描述组织当前采用的长期协作规范及其理由、边界和例外的权威载体。Handbook 不保存所有项目事实，也不是一次性通知和工具教程的集合。

## 单一事实源

某项当前事实或规则的权威维护位置。单一事实源不要求所有信息存放在同一个系统，而是要求发生冲突时能够判断哪里为准。参见[信息源与知识库架构](04-knowledge-base-architecture.md)。

## 权威载体

负责保存某类当前信息并接受更新的位置。它可以是文档、任务、代码仓库、决策记录或其他系统，具体形式由信息用途决定。

## DRI

Directly Responsible Individual，直接负责人。DRI 负责推动一个问题获得结论，并在必要输入和权限范围内作出或促成决定。DRI 不代表独自完成全部工作，也不扩大其合法权限。参见[决策记录与责任归属](10-decision-records.md)。

## Owner

负责促使某项长期信息、机制或资源保持可用的人或角色。Owner 可以组织他人共同维护，不等同于所有工作的执行者。只有责任归属无法从工作关系中判断时，才需要显式标注 Owner。

## 决策记录

保存一个重要决定的结论、责任归属、关键背景和取舍，使决定可以被执行、复查和合理修改的记录。它可以是独立文档，也可以存在于项目、任务或代码评审中。

## 里程碑

项目中用于观察结果、检查风险并重新决定是否继续或调整的判断点。里程碑不只是日期，也不等同于一组任务完成百分比。参见[项目计划与里程碑](08-project-milestones.md)。

## 规范正文

描述跨平台协作要求和预期结果的内容，是义务的来源。与之相对，平台落地只说明如何在某种工具中实现这些要求，不应新增义务。

## Skill

帮助 Agent 完成特定类型任务的一组说明。Handbook Skill 负责指导 Agent 查找、理解和引用 Handbook，不复制规范正文，也不维护另一套隐藏规则。
